# CS 230 Portfolio Submission

## Artifact
Software Design Document – The Gaming Room: Draw It or Lose It

## Reflection

The Gaming Room requested a redesign of their Android game, *Draw It or Lose It*, into a web-based application that supports multiple platforms and concurrent users. The goal was to ensure scalable, maintainable software that enforces unique names for games, teams, and players while allowing only one game instance in memory at a time.

In developing the software design document, I focused on clearly documenting system requirements, architecture, and design patterns. I feel I did particularly well translating the client’s needs into structured modules using object-oriented principles like inheritance, encapsulation, and design patterns such as Singleton and Iterator. This helped me plan the code effectively before implementation.

Working through the design document was helpful because it forced me to think about scalability, memory management, distributed systems, and cross-platform compatibility in advance. If I could revise one part, I would expand the domain model and include more detailed UML diagrams to clarify interactions between Game, Team, and Player objects.

To interpret the user’s needs, I carefully analyzed their requirements for multi-team gameplay, unique identifiers, and web-based access. Considering the user’s needs was critical because it guided my design choices for usability, reliability, and security. My approach involved modular design, defining workflows, and using object-oriented patterns to ensure the system is flexible and maintainable. In the future, I would continue using UML diagrams, iterative prototyping, and structured design documentation to guide software development.

Overall, this project reinforced the importance of planning architecture, enforcing uniqueness constraints, managing distributed interactions, and securing user data. The Linux server platform with Java, relational databases, and web-based client support was chosen as the recommended architecture to ensure scalability, performance, and cross-platform compatibility.
